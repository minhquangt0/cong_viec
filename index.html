  • Frontend：Next.js + React（Partner/User/Admin）
  • DB：PostgreSQL（必要に応じてRedis）
  • Video：AWS MediaConvert（HLS生成・トランスコード）、S3保管、CloudFront配信

  • Mailroom：Atena（紙契約の配送/受領・運用連携）
  • Jobs：SQS（キュー） / Sidekiq（Railsジョブ実行）（もしくはRails ActiveJob + Queue）
  • Auth：LINEログイン + JWT / セッション、RBAC（Role-Based Access Control）必須

8) Assumptions（前提条件）

8.1 アーキテクチャ/技術基盤

  • 既存MooMの共通コンポーネント/画面資産を活用できる（再利用率が一定以上）。
  • CM/VP/CPは「1つのPartnerアプリ」でRBAC（Role-Based Access Control）により表示/権限を切り替える。
  • 本システムはWebアプリケーションとして提供し、iOS/Androidネイティブアプリは対象外。
  • レスポンシブ対応はUser（入居者）画面のみ。Admin/Partner画面はPC（デスクトップ）前提。

8.2 外部連携/RPA

  • Itandi連携はRPA/スクレイピング前提。画面変更に伴う修正は都度発生しうる。
  • Itandiからのデータ同期はリアルタイム保証なし（バッチ/ポーリング前提）で、更新反映に5分以上の遅延が発生する場合がある。
  • Itandi RPA用アカウント（ID/パスワード）は貴社側で準備・管理する。
  • SNS投稿/指標取得は、各プラットフォームの審査・利用制限・仕様変更の影響を受ける。
  • SNS API の利用申請・審査は貴社名義で実施される（開発者アカウントを提供いただく）。
  • 外部API（Itandi/SNS/Atena等）のレート制限・ダウンタイムによるシステム影響は免責とする。
  • 外部サービス（Atena、周辺情報API等）の契約/アカウント準備は貴社側で実施。
  • Atena連携は「配送/受領ステータス同期・追跡番号管理・書類アップロード/閲覧」まで（OCR自動判定・自動紐付け・運用代行は対象外）。
  • Atena API 仕様書・認証情報は開発開始前に提供される。

8.3 動画/メディア

  • 録画データは、運用上の保管期間/容量上限/削除ポリシーを別途定義する。
  • 動画フォーマットは MP4/MOV を想定。その他の形式は事前変換が必要。
  • 1動画あたりの最大ファイルサイズは 150MBとする（超過時は分割が必要）。
  • 月間想定：物件登録500件、内覧予約2,000件、動画アップロード500本。

8.4 AI Chat Support

  • AI Chat Supportは「FAQ + RAG + ルーティング + 人手引継ぎ」まで。業務判断の自動化やAIエージェント化（予約/申込/契約操作）は対象外。
  • FAQ データ（Q&Aセット）は貴社側で初期データを提供する。
  • RAG 用の学習データ（過去チャットログ/マニュアル等）は貴社側で準備する。
  • AI の回答精度は80%程度を目標とし、100%保証はしない。
  • 不適切な回答が発生した場合の改善は別途チューニング作業が必要。

8.5 開発体制/プロセス

  • 週次の定例MTG（1時間）を実施し、進捗報告・課題共有・仕様確認を行う。
  • 仕様変更はMonth 4 までに確定し、それ以降の大幅変更は別途見積・工期調整が必要。
  • UAT は Month 7 に実施し、貴社側でテストケース作成・実施リソース（1-2名/2週間）を確保する。
  • 本番リリースは Month 8 末を想定するが、UAT結果により延期の可能性がある。

8.6 データ/環境

  • 本番環境/ステージング環境/開発環境の AWS アカウントは貴社側で事前に準備済み。

8.7 パフォーマンス/スケール

  • 同時接続ユーザー数は参考値として最大100名を想定（保証なし）。
  • 性能（応答速度/処理能力）は定量保証・SLAなし。性能試験（Load/Stress/Soak）は対象外。
  • マーケット拡大等により想定（同時接続100名/トラフィック）を超える場合の性能試験実施およびインフラ増強（スケール設計/チューニング）>は、別途見積・別途費用で対応する。

9.6 セキュリティ/品質保証

  • セキュリティは基本対応（OWASP相当の一般的対策）まで。高度なセキュリティ要件（脆弱性診断/ペネトレーションテスト/ゼロトラスト設計等）は対象外。

  • 性能試験（Performance Test / Load Test / Stress Test）は対象外。必要な場合は別途計画・見積。
  • DR（災害復旧）/ マルチリージョン構成 / 厳格な高可用性SLA（99.9%超など）の設計・実装は対象外（別途）。
