🚨 重要な前提条件・制限事項（フェーズ1：MVP）

  • MVP開発方針：
    • ✅ 基本機能の構築に集中（ベストエフォート対応）
    • ✅ パフォーマンス要件は実運用データ収集後に最適化（フェーズ2）
    • ✅ Terraformは実運用安定後に導入（フェーズ2）
    • ✅ オートスケーリングは需要確認後に実装（フェーズ2）
  • 確定技術スタック：
    • ✅ バックエンド：Python + FastAPI + SQLAlchemy
    • ✅ データベース：RDS MySQL 8.0
    • ✅ 認証：Amazon Cognito + JWT
    • ✅ インフラ：AWS Fargate (Container)
    • ✅ ストレージ：S3 (音声ファイル30日間保存)
    • ✅ TTS：Style-Bert-VITS2（日本側提供）
  • パフォーマンス前提（MVP）：
    • ✅ 想定負荷：約20同時リクエスト
    • ✅ 音声長：最大60秒（800-1000文字）
    • ✅ レスポンスタイム：短文1-2秒（目安）
    • ✅ SLA：99.5%（MVP初期運用）
    • ⚠️ 負荷試験・チューニングはフェーズ2で実施
  • データ保存ポリシー：
    • ✅ S3音声ファイル：30日間保存後自動削除
    • ✅ ユーザーデータ：退会後30日間保持後削除
    • ✅ 生成数制限：1日50回（MVP仮設定）
  • テスト戦略：
    • ✅ 各機能の15%工数 = 単体・コンポーネントテスト（120.75h）
    • ✅ 結合テスト（60h）= E2Eシナリオテスト
    • ✅ セキュリティ試験（40h）= OWASP Top10・基本診断
    • ❌ 負荷試験はフェーズ2で実施
  • UAT期間：2週間〜4週間（フィードバック反映含む）
  • 保守運用：β版リリース後3ヶ月間の無償保守（バグ修正のみ）
  • 運用監視：CloudWatch自動アラート設定のみ（24/7監視は別途）
