委託者（以下「甲」という。）と受託者（以下「乙」という。）は、2025年6月23日付で締結した「MatchHub+ARMSシステムの開発業務委託契約書」（以下「原契約」という。）について、開発要件の追加及び仕様変更に伴い、以下のとおり契約内容の変更について合意する。

第1条（変更の背景）

原契約の履行過程において、システムの機能拡張及び追加要件が発生したため、開発範囲、金額及びスケジュールを見直すこととなった。本覚書は、かかる変更内容を明確化し、双方の合意事項を確認するものである。

第3条（開発スケジュールの変更）

原契約における開発期間を以下のとおり変更する。

1. 原契約における開発期間：2025年6月25日から2025年12月15日まで。

2. 詳細な開発スケジュールは、第4条に定めるとおりとします。

3. 受入検収期間は、各フェーズの成果物の納入日から起算して30日間とする。

第4条（開発フェーズの分割）

システム開発を以下の2フェーズに分けて実施する。

1. Phase 1（コア機能の実装）

  • 開発時期：別紙の見積書(システム開発)
  • 開発期間：原則として3ヶ月（または別紙1のスケジュールに従う）
  • 受入検収期間：各フェーズのステージング環境へ納品した時点（乙の納品完了通知時点）から起算して30日間とする。
  • 委託料：金1,818,000円（税抜き）
  • 実施内容：公式サイト、CA及び求職者サイト、Zキャリア、求人ボックス等に関する。画面設計・UI／UXデザインを含むコア機能の実装並びに基本アーキテクチャの構築
  • 本番公開：Phase 1検収完了後に開始する。
※注記：ベトナム側・日本側の年末年始の長期休暇を考慮し、5月末を目処にバッファを含めて設定。5月末にPhase 1検収完了及びエンドユーザー向け本番公開開始を想定。

2. Phase 2（拡張機能の実装）

  • 開始時期：別紙の見積書(システム開発)
  • 開発期間：原則として3ヶ月。
  • 受入検収期間：各フェーズのステージング環境へ納品した時点（乙の納品完了通知時点）から起算して30日間とする。
  • 委託料：金2,680,000円（税抜き）
  • 検収完了予定：2026年7月末
  • 実施内容：管理者機能、Google Jobs、engage、Indeed、SA用機能等の拡張機能の追加実装、並びに全体の仕上げ

3. Phase 1とPhase 2は、技術的な依存関係に応じて一部並行して進めることができる。

4. 各フェーズの詳細仕様は、 別紙の見積書(システム開発)に従う。

第5条の2（UI/UXデザインの前提及び進め方）

1. UI/UXデザインは、2025年9月に双方で確定した画面モックを前提として実施する。

2. 乙は、甲のデザイン要望（トーン＆マナー、参考デザイン、優先順位等）をヒアリングし整理する。

3. 乙は、代表画面1画面について2案のデザイン案を提示し、甲はそのうち1案を選定する。

4. 乙は、甲が選定したデザイン案を基準として、別紙の見積書(システム開発)に定める他画面のデザインを作成する。

5. 甲はデザインに対してフィードバックを行うことができ、修正ラウンドは最大3回とする。

第5条の3（仕様変更・追加要望の取扱い）

1. 文言修正、配置・余白等の外観調整など、業務フロー・画面遷移・データ項目・DB/API仕様に影響しない軽微な修正は、契約範囲内として対応する。

2. 前項に該当しない変更（新機能追加、業務要件の追加・変更、画面追加、データ項目追加、再設計・再実装を伴うもの等）は契約範囲外とし、乙は追加費用・納期影響を見積の上、甲乙協議により対応可否を決定する。

3. 追加対応は、甲の書面または電磁的記録（メール等）による承認をもって着手する。

4. 本システムは、Zキャリアおよび各種求人媒体／外部Webサイト（以下「外部サービス等」という）に依存する機能（データ連携、求人取得、情報参照等）を含む。外部サービス等の仕様変更、画面構造変更、認証方式・利用規約の変更、アクセス制限（ブロック／レート制限／CAPTCHA等）、提供停止、機能制限、または非対応等により、乙（AMCOLAB）が当初想定した方法での取得・連携が困難または不可能となった場合、乙は速やかに甲へ報告し、代替手段（方式変更、対象範囲変更、運用回避策等）を甲乙協議の上で決定する。前項の事由により追加対応（調査、設計変更、実装変更、再実装、検証追加等）が必要となる場合、当該対応は原則として契約範囲外（追加開発）とし、乙は追加工数・費用およびスケジュール影響を見積の上、甲の承認を得て対応する。なお、外部サービス等は乙の管理外であるため、外部サービス等の変更・制約に起因する影響について、乙は責任を負わないものとする（ただし、乙の実装不備に起因する場合を除く）。

第6条（追加機能の概要）

本変更により追加される主な機能は以下のとおりである。本見積および機能範囲の前提は、Figmaデザイン（2025年9月）とする。

カテゴリ 主な追加機能
管理者機能 報酬・支払管理、Finance Hub連携、Z キャリアRPA設定、Google for Jobs設定、求人ボックス設定、Indeed・engage連携、質問管理、CA管理、求人管理、候補者管理、応募管理、アカウント管理、報酬率設定
CA機能 ダッシュボード、候補者管理、推薦機能、面接設定、Google Calendar連携、報酬管理、メッセージ機能、求人管理、各種設定
SA機能 要支援候補者管理、CA機能と同様の機能
公式サイト トップページ、求人一覧・詳細、応募フォーム、応募者マイページ、職務経歴書AI生成、メッセージ画面、法人向けサービス、事例紹介、コラム、各種法定ページ
外部連携 Z キャリアRPA、Google for Jobs、求人ボックス、Indeed、engage、Finance Hub、Google Calendar API

本条は概要であり、各フェーズの範囲は別紙の見積書(システム開発)を優先する。

第7条（原契約の効力）

本覚書に定めのない事項については、原契約の定めに従うものとする。本覚書と原契約の内容が矛盾する場合は、本覚書の内容が優先するものとする。

第8条（覚書の成立）

本覚書は、甲乙両者が署名又は記名押印（電子的な署名又は押印を含む）した文書をPDF形式にて電子メールで相互に送受信すること、または、署名若しくは記名押印に代わる電磁的処理を施すことによって締結するものとする。甲及び乙は、電磁的記録（電子データ）を本覚書の原本とすることに同意し、それを各自保存するものとする。

本覚書の締結を証するため、本書を電磁的記録として作成し、甲乙はCloudSignを利用した電子署名をもって本覚書を締結するものとする。なお、本電磁的記録は、本覚書の原本と同一の効力を有するものとする。
